# UHPS-full — Hunmin의 진짜 핵심

> NIKL은 한국어 외래어 표기 표준일 뿐.  
> **UHPS-full은 한국 화자가 읽을 수 있는 IPA-faithful 음소 표기다.**

## 왜 UHPS-full인가

| 기준 | NIKL (HUNMIN-readable) | **UHPS-full** |
|------|----------------------|--------------|
| 한국 화자 가독성 | ✓ (한국어 음소만) | ✓ (옛한글 추정 가능) |
| IPA 정보 보존 | × (음소 손실 큼) | **✓ 95%+** |
| /f/ /v/ /z/ /θ/ /ð/ 변별 | × (모두 머지) | **✓ ㆄ ㅸ ㅿ ㅼ ㅽ** |
| 장단/강세 보존 | × | **✓ ː ·** |
| 성조 (Mandarin/Vietnamese) | × | **✓ 〮 〯 ↗ ↘** |
| 위원회 결정 의존 | ✓ (수많은 단어별 예외) | **× 룰만** |
| ML 음소 공간 통일 | × (NIKL = lossy) | **✓** |

## "한국 화자가 읽을 수 있는가" — 검증

UHPS-full은 옛한글을 사용하지만 한국 화자가 보면 자연스럽게 추정 가능:

| 옛한글 | 현대 음소로 추정 |
|--------|----------------|
| ㆄ | "조금 강한 ㅍ" → /f/ |
| ㅸ | "조금 약한 ㅂ" → /v/ |
| ㅿ | "조금 약한 ㅈ" → /z/ |
| ㆅ | "강한 ㅎ" → /x/ |
| ᄾ | "치찰 ㅅ" → /ʃ/ |
| ᄶ | "치찰 ㅈ" → /ʒ/ |
| ᄛ | "가벼운 ㄹ" → /ʁ/ |
| ㅼ | "ㅅ+ㄷ" → /θ/ |
| ㅽ | "ㅅ+ㅂ" → /ð/ |
| ㆎ | "ㅗ+ㅓ 사이" → /ɔ/ |
| ㆍ | "ㅏ+ㅓ 사이" → /ɑ/ |
| ㆁ | "ㅇ 받침 강조" → /ŋ/ |
| ㆆ | "ㅎ+ㄱ 시작" → /ʔ/ |
| · | "강세" |
| ː | "장음" |

이 매핑은 직관적이라 한국어 화자 누구나 5-10분 학습 후 UHPS-full 자연스럽게 읽음.

## 실제 사용

```python
from hunmin import transcribe, UHPS_FULL

transcribe('Hello, world!', 'en', mode=UHPS_FULL)
# → '허로·우 으ː륻'

transcribe('Mozart', 'de', mode=UHPS_FULL)
# → '모·ː차ㄹ트'  (강세 위치 명확)

transcribe('father', 'ipa', mode=UHPS_FULL)  
# IPA 직접 입력
transcribe('ˈfɑːðɚ', 'ipa', mode=UHPS_FULL)
# → 'ㆄㆍː ㅽ어'
```

## 왜 NIKL은 부수적인가

NIKL 외래어 표기법:
- **위원회 결정의 누적** — 룰이 아니라 단어별 정착
- **음소 손실 적극 허용** — 한국어 음절 구조 우선
- **단어별 이질성** — z가 ㅈ/ㅉ/ㅊ로 다르게 표기
- **OCR 오염 영향** — 외래어 표기 용례집 PDF에 다이어크리틱 깨짐

NIKL은 "한국어로 정착된 표기" 측정. UHPS-full은 "음소 정보 보존" 측정. **목적이 다름**.

## 결론
